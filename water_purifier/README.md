# 净水机刷卡控制系统

基于 Arduino 的净水机刷卡控制方案。用户刷卡后自动控制净化水或热水出水，流量计精确计量，达到目标量自动停止。

## 功能

- **刷卡出水** — 刷授权卡自动出水，不同卡绑定不同出水类型
- **净化水 / 热水** — 两种出水模式，独立电磁阀/水泵控制
- **精确计量** — 霍尔流量计（YF-S201）计量出水量，到量自停
- **恒温控制** — DS18B20 测温 + 继电器控制加热器，带回差恒温（目标 92℃）
- **多重保护** — 出水超时（60s）、断流检测（2s）、加热超时（5min）、RFID 自检
- **状态显示** — LCD1602 I2C 显示温度、出水量、系统状态
- **声光提示** — 蜂鸣器 + 4 路 LED 指示刷卡和运行状态

## 硬件清单

| 模块 | 型号 | 数量 |
|------|------|------|
| 主控板 | Arduino Uno R3 / Mega 2560 | 1 |
| 读卡器 | RFID-RC522 (MFRC522) | 1 |
| 温度传感器 | DS18B20 防水探头 + 4.7kΩ 电阻 | 1 |
| 流量计 | YF-S201 霍尔流量计 | 1 |
| 显示屏 | LCD1602 I2C (地址 0x27 或 0x3F) | 1 |
| 继电器模块 | 5V 双路/四路继电器 (低电平触发) | 1 |
| 蜂鸣器 | 有源蜂鸣器 5V | 1 |
| LED | 5mm LED × 4 + 220Ω 限流电阻 | 4 |
| 卡片 | S50 白卡 / 钥匙扣 (13.56MHz Mifare) | 若干 |

## 引脚接线

```
Arduino         ->  外设
=============================
D10 (SS)        ->  RFID-RC522 SDA
D9              ->  RFID-RC522 RST
D11 (MOSI)      ->  RFID-RC522 MOSI
D12 (MISO)      ->  RFID-RC522 MISO
D13 (SCK)       ->  RFID-RC522 SCK

D5              ->  继电器1 (净化水阀/泵)
D6              ->  继电器2 (热水阀/泵)
D7              ->  继电器3 (加热器)

D3              ->  DS18B20 DATA (须接 4.7kΩ 上拉至 VCC)
D2              ->  流量计信号线 (中断)

A4 (SDA)        ->  LCD I2C SDA
A5 (SCL)        ->  LCD I2C SCL

D8              ->  蜂鸣器 (+)
D13             ->  LED 就绪指示灯 (Uno 板载 LED, SPI 通信时会闪烁)
A0              ->  LED 净化水指示
A1              ->  LED 热水指示
A2              ->  LED 故障指示
```

> **注意**: Uno 的 D13 同时是 SPI-SCK 和板载 LED，RFID 通信时 LED 会闪烁，属正常现象。
> Mega 2560 的硬件 SPI 在 50-53 脚，D13 不受影响。

## 依赖库

在 Arduino IDE 中打开「项目」→「加载库」→「管理库」，搜索并安装：

| 库名 | 作者 | 用途 |
|------|------|------|
| **MFRC522** | GithubCommunity | RFID 读卡 |
| **OneWire** | Paul Stoffregen | DS18B20 单总线 |
| **DallasTemperature** | Miles Burton | DS18B20 温度读取 |
| **LiquidCrystal_I2C** | Frank de Brabander | LCD1602 I2C 显示 |

## 快速上手

1. 用 Arduino IDE 打开 `净水机.ino`
2. 在库管理器中安装上述 4 个依赖库
3. 选择正确的开发板（Uno / Mega 2560）和端口
4. **确认 LCD I2C 地址**: 如果不显示，用 I2C 扫描器确认地址，修改 `config.h` 中的 `LCD_I2C_ADDR`（常见: `0x27` 或 `0x3F`）
5. 编译并上传

## 刷卡操作

| 卡类型 | 行为 |
|--------|------|
| 管理员卡 | 双短蜂鸣确认 (配置模式预留) |
| 净化水卡 | 出 500ml 净化水 |
| 热水卡 | 出 300ml 热水 (须水温 ≥ 85℃) |
| 通用卡 | 出 500ml 净化水 |
| 未授权卡 | 两声长蜂鸣拒绝 |

## 加热控制

- 自动加热: 水温 < 89℃ (92℃目标 - 3℃回差) 时启动加热
- 自动停止: 水温 ≥ 92℃ 时停止
- 加热超时: 超过 5 分钟持续加热则报警
- 传感器故障: 返回 -127℃ 时自动跳过加热控制

## 安全保护

| 保护项 | 条件 | 动作 |
|--------|------|------|
| 出水超时 | 连续出水超过 60 秒 | 关闭阀门，蜂鸣报警，进入错误状态 |
| 断流检测 | 启动后 2 秒内无流量脉冲 | 关闭阀门，进入错误状态（水桶空/管路堵） |
| 加热超时 | 连续加热超过 5 分钟 | 关闭加热器，进入错误状态 |
| RFID 故障 | 启动时自检失败 | 系统锁定，闪烁故障灯 |
| 错误恢复 | 错误持续 10 秒 | 自动清除错误，恢复待机 |

## 定制修改

- **修改出水量**: 在 `config.h` 中修改 `DEFAULT_PURE_WATER_ML` / `DEFAULT_HOT_WATER_ML`
- **添加/删除用户卡**: 在 `config.h` 中修改 `USER_CARDS` 和 `USER_CARD_TYPES` 数组
- **修改热水温度**: 改 `HOT_WATER_TARGET_TEMP` 和 `HOT_WATER_MIN_TEMP`
- **修改流量计系数**: 不同流量计脉冲数不同，改 `PULSES_PER_LITER`
- **修改 LCD 地址**: 改 `LCD_I2C_ADDR` (扫描器确认后)
- **修改单次最大出水量**: 改 `MAX_WATER_ML`

## 首次使用：录入新卡

1. 用串口监视器 (9600 波特率) 连接 Arduino
2. 刷一张新卡，串口会打印卡 UID（如 `检测到卡: AB:CD:EF:02`）
3. 将该 UID 的 4 字节写入 `config.h` 的 `USER_CARDS` 数组
4. 在 `USER_CARD_TYPES` 对应位置设置类型: 0=净化水, 1=热水, 2=均可
5. 重新编译上传
